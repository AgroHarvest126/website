# AgroHarvest — сайт

## Структура проекта
```
agroharvest-site/
├── index.html              ← вся вёрстка, стили и скрипты (один файл)
├── assets/
│   ├── model/
│   │   └── model.glb        ← 3D-модель устройства (hero, шаги, контакты)
│   └── video/
│       ├── field.webm        ← карточка "Полевые культуры"
│       ├── greenhouse.webm   ← карточка "Теплицы"
│       ├── polig.webm        ← карточка "Научные полигоны"
│       ├── iot.webm          ← шаг "Генерация" (Как работает технология)
│       └── lora.webm         ← шаг "Передача" (Как работает технология)
└── README.md
```

## Как выложить на GitHub Pages

1. Создайте новый репозиторий на GitHub (например, `agroharvest-site`).
2. Загрузите в него содержимое этой папки **как есть**, сохранив структуру (`index.html` и `assets/` — в корне репозитория).
   - Через веб-интерфейс: "Add file → Upload files", перетащите всю папку.
   - Или через git:
     ```bash
     git init
     git add .
     git commit -m "AgroHarvest site"
     git branch -M main
     git remote add origin https://github.com/<ваш-логин>/agroharvest-site.git
     git push -u origin main
     ```
3. В репозитории: **Settings → Pages**.
4. В разделе "Build and deployment" → Source: **Deploy from a branch**.
5. Branch: **main**, папка: **/ (root)** → Save.
6. Через 1–2 минуты сайт будет доступен по адресу:
   `https://<ваш-логин>.github.io/agroharvest-site/`

## Важно про видео и 3D-модель

- Все `.webm` подключены как фоновые зацикленные видео (`autoplay muted loop playsinline`) — работает без плеера и звука, как в hero Apple/Tesla.
- 3D-модель `model.glb` рендерится через `<model-viewer>` (веб-компонент Google, подключается из CDN в `<head>`) — даёт вращение мышью/пальцем и авто-поворот без написания WebGL-кода.
- GitHub Pages отдаёт `.webm` и `.glb` как обычные статические файлы — ничего дополнительно настраивать не нужно, но если репозиторий приватный, включите Pages только на публичном репозитории (приватные Pages требуют GitHub Pro/Team).

## Чего пока не хватает (см. комментарии в коде `index.html`)

- Видео/фото для карточки **"Виноградники и сады"** — сейчас там плейсхолдер, ищите `<!-- IMAGE/ВИДЕО: пока нет ассета -->` в коде.
- Реальные цифры в разделе **"Характеристики"** вместо `[X]` — обязательно заменить перед показом инвестору/фонду.
- Кастомная line-art диаграмма архитектуры (сейчас есть рабочая HTML-версия с анимацией, плюс место под доп. иллюстрацию для презентаций/PDF).

## Локальная проверка перед публикацией

Откройте `index.html` через локальный сервер (не просто двойным кликом — некоторые браузеры блокируют загрузку `.glb`/`.webm` по `file://`):

```bash
cd agroharvest-site
python3 -m http.server 8000
```

Затем откройте `http://localhost:8000` в браузере.
